# File properties

::: guidance
This chapter covers topics that you will encounter frequently when using the command line in Linux. It is therefore worth paying extra attention to this chapter.
:::

Make sure you are in your home directory.

Type the command `ls -la`. Note the space between the command (`ls`) and the arguments (`-la`).

You can see much more information than with the plain `ls` command alone. We'll highlight one row for closer inspection.

``` txt
 III         II                        I
drwxrwxr-x 2 __USERNAME__ __USERNAME__ 4096 Apr  1 13:35 noppe-cli
```

Let's go through them (`I - III`) in the following chapters.

## I Metadata

The end of the row contains information such as

   1. the file's size in bytes,
   2. its last modification date,
   3. and its name.

::: note
The very first character in the row indicates the file type (`d` for directory).
:::

Use the command `echo "This file contains text" > textfile` to create a file.

Use the command `ls -la` to verify that the file was created. Pay attention to the file's timestamp.

Next, execute the `mv textfile file-with-text` command.

Then, use the `ls -la` command again to check that `textfile` has been renamed to `file-with-text`. Compare the timestamp with the previous value.

::: tip
To be really sure that it is the same file, you can use the command `cat file-with-text` to check the contents if it.
:::

## II User and group

Type the command `id`. The command prints the current user's user ID and group IDs.

:::terminal
__USERNAME__@__HOSTNAME__:~$ id
uid=1001(__USERNAME__) gid=1001(__USERNAME__) groups=1001(__USERNAME__)
__USERNAME__@__HOSTNAME__:~$ █
:::

Also, each file is assigned to a user and a group. These assignments are listed in between the metadata and permissions.

## III Permissions

Now, let's take a closer look at the file permissions. First, let's reformat the previous printout slightly differently:

```txt
   U   G   O
d rwx rwx r-x
```

The set of permissions is repeated three times. The first set relates to the permissions assigned to the user (`U`) for the file. The second set is for the group (`G`), and the third set is for other (`O`) users of the computer. The permissions are:

| symbol | permission | octal |
|-----|------|-----|
| `r` | read | `4` |
| `w` | write | `2` |
| `x` | execute | `1` |
| `-` | no permission | `0` |

Execute the command `ls -l file-with-text`. Pay attention to the permissions and file's user and group.

Then, execute the command `chmod g-w,o-r file-with-text` and check the changes to the permissions with the ls command above.

The 'chmod' command allows you to add or remove permissions for the file owner, group, or other users. Multiple modifications can be made at once, separating them with commas.

::: terminal
__USERNAME__@__HOSTNAME__:~$ ls -l file-with-text
-rw-rw-r-- 1 __USERNAME__ __USERNAME__   24 Apr  6 14:28 file-with-text
__USERNAME__@__HOSTNAME__:~$ chmod g-w,o-r file-with-text
__USERNAME__@__HOSTNAME__:~$ ls -l file-with-text
-rw-r----- 1 __USERNAME__ __USERNAME__   24 Apr  6 14:29 file-with-text
__USERNAME__@__HOSTNAME__:~$ █
:::

Let's have another example. Type the command `echo "For your eyes only" > access-denied`. Verify, that you can see the content of file with the `cat access-denied` command and check the default file permissions with the `ls -l access-denied` command.

Then, execute the command `chmod 007 access-denied`. What do you think will happen? Take another look at the table above.

Try out the changed permissions using the commands `echo hi > access-denied` and `cat access-denied`.

::: terminal
__USERNAME__@__HOSTNAME__:~$ echo "For your eyes only" > access-denied
__USERNAME__@__HOSTNAME__:~$ cat access-denied
For your eyes only
__USERNAME__@__HOSTNAME__:~$ ls -l access-denied
-rw-rw-r-- 1 __USERNAME__ __USERNAME__   24 Apr  6 14:32 access-denied
__USERNAME__@__HOSTNAME__:~$ chmod 007 access-denied
__USERNAME__@__HOSTNAME__:~$ echo hi > access-denied
-bash: access-denied: Permission denied
__USERNAME__@__HOSTNAME__:~$ cat access-denied
cat: access-denied: Permission denied
__USERNAME__@__HOSTNAME__:~$ ls -l access-denied
-------rwx 1 __USERNAME__ __USERNAME__   24 Apr  6 14:33 access-denied
__USERNAME__@__HOSTNAME__:~$ chmod u+rw,g+rw,o-wx access-denied█
:::

::: guidance
A user assigned to a file can remove their own permissions to read, write or execute the file, but can also restore them if desired.
:::

Finally, try restoring the original permissions (`rw-rw-r--`) for the `access-denied` file.

::: note
At some point the <span class="text-green">access-denied</span> file got a nice green colour in the `ls` print. This means that the file is executable, which is not always a good thing. You can find more information in the [Executables](04-executables.md) chapter.
:::
