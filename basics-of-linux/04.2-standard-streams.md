# Standard streams

When a program is executed in Linux, three streams are typically opened for it when it starts up. One is for input, one is for output, and one is for printing diagnostic or error messages. Usually all three are attached to the user's terminal.

::: service
File descriptors (fd) are not covered in this Basics of Linux course. However, it is useful to know that stdin has fd `0`, stdout has fd `1`, and stderr has fd `2`, as you will see later on.
:::

## Input (stdin)

The input stream is referred to as "standard input". A typical example of standard input would be a keyboard, but it can also be the output of another command.

Let's give this a try with a simple Bash script:

```bash
#!/bin/bash

read -r input

echo "Input was: $input"
```

Create the script `repeater` using the `cat << 'EOF' > repeater` command or a text editor. Make the script executable.

::: tip
Please note that you must use hyphens around `'EOF'` since we are using a variable in our input. 
:::

Run the script and enter some text, such as `Hello World`. In this case, the keyboard entries act as standard input (stdin) for the script.

::: terminal
{{HOSTNAME}}@{{USERNAME}}:~$ cat << 'EOF' > repeater
&gt; #!/bin/bash
&gt; 
&gt; read -r input
&gt;
&gt; echo "Input was: $input"
&gt; EOF
{{HOSTNAME}}@{{USERNAME}}:~$ chmod +x repeater
{{HOSTNAME}}@{{USERNAME}}:~$ ./repeater
Hello World
Input was: Hello World
{{HOSTNAME}}@{{USERNAME}}:~$ █
:::

## Output (stdout)

The output stream is referred to as "standard output". It typically prints to the user's terminal.

For example, if you type the command `echo Hello World`, the text `Hello World` is written in the standard output.

The output of one command can be used as the input for another. Try running the `repeater` script with the command `echo Hello World | ./repeater`.

::: service
Another command's output as an input is handled more thoroughly in the [Redirections](08-redirections.md) chapter.
:::

::: terminal
{{HOSTNAME}}@{{USERNAME}}:~$ echo Hello World
Hello World
{{HOSTNAME}}@{{USERNAME}}:~$ echo Hello World | ./repeater
Input was: Hello World
{{HOSTNAME}}@{{USERNAME}}:~$ █
:::

## Error (stderr)

The error stream is referred to as 'standard error' and behaves similarly to standard output.

For example, if you type the command `cat noexist`, the error message for a non-existent file is written in the standard error.

To demonstrate that the stdout and stderr are a separate stream, run the command `cat noexist | ./repeater`.

::: guidance
So, by default, the `stdout` of the command is connected to the `stdin` of the other program.
:::

::: terminal
{{HOSTNAME}}@{{USERNAME}}:~$ cat noexist
cat: noexist: No such file or directory
{{HOSTNAME}}@{{USERNAME}}:~$ cat noexist | ./repeater
cat: noexist: No such file or directory
You wrote:
{{HOSTNAME}}@{{USERNAME}}:~$ █
:::

::: service
Remember `stdout` being number `1` and `stderr` being number `2`? Try out the following command:  
&nbsp;&nbsp;&nbsp;`cat noexist 2>&1 | ./repeater`  
In this command, stderr (`2`) is redirected (`>`) into the command's stdout (`&1`), which is then routed to (`|`) other command's stdin (`0`).
:::

::: terminal
{{HOSTNAME}}@{{USERNAME}}:~$ cat noexist 2>&1 | ./repeater
You wrote: cat: noexist: No such file or directory
{{HOSTNAME}}@{{USERNAME}}:~$ █
:::
